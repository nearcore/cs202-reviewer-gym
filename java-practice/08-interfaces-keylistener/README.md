# Interfaces and KeyListener Practice

## Topic
Interface contracts, `implements`, `@Override`, keyboard events, focus, and repainting.

## Goal
Implement behavior a class promises and move a Swing square with arrow keys.

## Instructions
Run `MoveSquareGame.java`, click the panel if it needs focus, and use the arrow keys.

## Starter Code
`starter/DrawableStarter.java`

## Expected Output
The mini project opens a window with a moveable square.

## Questions Before Running
1. Why are all three KeyListener methods required?
2. Which key code moves right?
3. Why call repaint after changing x or y?

## Reflection
1. Did focus affect input?
2. What topic does this connect to? Graphics and overriding.
