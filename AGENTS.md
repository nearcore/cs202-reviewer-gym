# AGENTS.md

## Project Goal

This project is a CS202 Java Reviewer Gym. It helps a Java student learn by active practice: explain, trace, fix, and code.

## Main Rule

Do not create passive notes only. Every topic must include practice.

## Tone

Use plain, student-friendly explanations. Avoid robotic textbook language. Be clear, direct, and practical.

## Code Style

- Keep HTML, CSS, and JavaScript simple and dependency-free.
- Keep Java examples beginner-friendly and runnable on Java 21+ where practical.
- Prefer short, focused Java files.
- Add comments only when they help learning.
- Do not over-engineer.

## Required Learning Pattern

For every topic, include:

1. Simple explanation
2. Short Java example
3. Trace-the-output practice
4. Bug-fixing practice
5. Small coding task
6. Vocabulary or concept check
7. Final-exam-style question

## Java Practice Rule

Every major topic needs a runnable Java practice file and, when useful, a matching solution file. Broken-code files are deliberately non-compiling and must say so in their README.

## Do Not

- Do not make the app only decorative.
- Do not use vague explanations.
- Do not skip exercises.
- Do not hide answers without a way to review them.
- Do not make setup complicated.
